## 1. Project Overview
- **Objective:** Fine-tune a Small Language Model (SLM), specifically **Qwen 2.5 7B Instruct**, to act as a virtual capstone project supervisor.
- **The "Socratic Mentor" Persona:** The AI must NEVER spoon-feed code to students. It must guide them through technical hurdles, architecture design, troubleshooting, and Viva (oral exam) preparation by asking clarifying questions, providing hints, and debugging errors based on the curriculum.
- **Scope:** The model needs to be an expert on a 300-hour capstone curriculum and 60 specific Capstone Projects across AI, Cyber Security, ERP, and IoT domains.

## 2. Technical Stack & Pipeline
- **Base Model:** Qwen 2.5 7B Instruct
- **Fine-Tuning Method:** QLoRA using the highly optimized **Unsloth** library (to reduce VRAM usage and double training speed).
- **Compute Environment:** We successfully prototyped the fine-tuning pipeline on Kaggle's free T4 GPU (14.5 GB VRAM). However, due to Kaggle's 12-hour session limits and 20GB disk limits, the final production training run of 11,000 samples may be moved to a dedicated physical workstation (e.g., RTX 3090/4090).
- **Dataset Format:** Strict JSONL format mimicking ChatML structure:
  `{"messages": [{"role": "system", "content": "..."}, {"role": "user", "content": "..."}, {"role": "assistant", "content": "..."}]}`

## 3. Dataset Requirements (Production Goal)
We are currently in **Phase 4**, building a master `train.jsonl` dataset with **~11,000 samples**.
- **Project-Specific:** 9,000 samples (150 samples for each of the 60 projects).
- **Context Discovery:** 500 samples (teaching the AI to ask clarifying questions).
- **General Curriculum:** 1,000 samples (teaching the 300-hour syllabus).
- **General Persona:** 500 samples (Socratic teaching style).

## 5. Critical Technical Constraints
- **System Prompt Injection:** It is absolutely critical that every single JSON line generated by Claude includes the specific `[PROJECT CONTEXT]` block inside its `"system"` role message. Without this, the fine-tuned model won't learn the specific architectures, modules, and troubleshooting steps of the 60 projects.
- **Exporting:** Once training is done in Phase 5, the model will be exported as a 4-bit `GGUF` file (`q4_k_m`) for local inference via Ollama or a similar UI.
